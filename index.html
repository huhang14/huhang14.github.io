1.4 Padding
发表于2021-10-07|深度学习|吴恩达CNN
假设输入图片的大小为 n×n，而滤波器的大小为 f×f，则卷积后的输出图片大小为 (n−f+1)×(n−f+1)。 这样就有两个问题： 每次卷积运算后，输出图片的尺寸缩小； 原始图片的角落、边缘区像素点在输出中采用较少，输出图片丢失边缘位置的很多信息。 为了解决这些问题，可以在进行卷积操作前，对原始图片在边界上进行填充（Padding），以增加矩阵的大小。通常将 0 作为填充值。 设每个方向扩展像素点数量为 p，则填充后原始图片的大小为 (n+2p)×(n+2p)，滤波器大小保持 f×f不变，则输出图片大小为 (n+2p−f+1)×(n+2p−f+1)。 在进行卷积运算时，我们有两种选择： Valid 卷积：不填充，直接卷积。结果大小为 (n-f+1)×(n-f+1)； Same 卷积：进行填充，并使得卷积后结果大小与输入一致，所以需要 p=(f-1)/2。在计算机视觉领域，f通常为奇数。原因包括 Same 卷积中 p=(f-1)/2能得到自然数结果，并且滤波器有一个便于表示其所在位置的中心点。

1.2 卷积运算和边缘检测示例
发表于2021-10-07|深度学习|吴恩达CNN
卷积运算用一个正方形的过滤器（filter）对图像进行卷积运算。 将3x3的卷积核覆盖到图像左上的3x3区域上，然后用元素乘法把对应位置的数字相乘，最后把得到的9个数相加，填在4x4的第一格。 3x3的卷积核从左到右，由上到下每次移动一格，重复以上操作。 以上这个过程在 Python 中，卷积用conv_forward()表示；在 Tensorflow 中，卷积用tf.nn.conv2d()表示；在 keras 中，卷积用Conv2D()表示。 边缘检测 垂直边缘检测 水平边缘检测 垂直边缘检测例子 最左边的矩阵是图像，左边亮，右边暗 利用中间的3x3滤波器进行卷积运算 得到最右边矩阵，把它看作图像就是一条白色的柱
1.1 计算机视觉
发表于2021-10-07|深度学习|吴恩达CNN
一般的计算机视觉问题包括以下几类： 图片分类 目标检测 神经风格转换 应用计算机视觉时要面临的一个挑战是数据的输入可能会非常大。例如一张 1000x1000x3 的图片，神经网络输入层的维度将高达三百万，使得网络权重 W 非常庞大。这样会造成两个后果： 神经网络结构复杂，如果数据不够巨量，容易出现过拟合； 所需内存和计算量巨大。 因此，一般的神经网络很难处理蕴含着大量数据的图像。解决这一问题的方法就是使用卷积神经网络（Convolutional Neural Network, CNN）。
3.10 R-CNN（Region CNN）
发表于2021-10-07|深度学习|吴恩达CNN
滑动窗口目标检测算法对一些明显没有目标的区域也进行了扫描，扫描是低效的 R-CNN pick a few regions or a few windows to run conv classifier 使用图像分割算法构建了右图 在不同的色块上找出候选区域（Region Proposal） 只在Region Proposal run conv classifier 减少扫描的区域 缺点R-CNN 的缺点是运行速度很慢，所以有一系列后续研究工作改进。 Fast R-CNN（与基于卷积的滑动窗口实现相似，但得到候选区域的聚类步骤依然很慢） Faster R-CNN（使用卷积神经网络来获取候选区域，比 Fast R-CNN快得多）。不过大多数时候还是比 YOLO 算法慢。
